## How Open Arcana works

Open Arcana sits between Claude Code and your Obsidian vault. It configures Claude Code's behavior through three mechanisms:

1. **Rules** (`.claude/rules/*.md`): Loaded into context on every session. They shape how the AI thinks, reasons, and validates its work.
2. **Hooks** (`.claude/hooks/*.sh`): Shell scripts triggered by Claude Code events (SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, PreCompact, Stop). They enforce constraints and inject context.
3. **Commands** (`.claude/commands/*.md`): Slash commands available during sessions. Each one is a prompt template that the user invokes with `/command-name`.

## File ownership (quick reference)

Open Arcana creates files only in these locations (see `../ARCHITECTURE.md` § File ownership for the full map):

- `.claude/rules/`, `.claude/hooks/`, `.claude/commands/`, `.claude/tools/`
- `.claude/settings.local.json`, `.claude/arcana.config.yaml`
- `CLAUDE.md` (vault root)
- `00-Dashboard/`, `80-Templates/`

Your notes, daily notes, and MOCs are never touched by setup or uninstall.

## Configuration

All user-specific values live in `.claude/arcana.config.yaml`, generated by `./setup.sh`. This file is in `.gitignore` by default. Template variables (`{{USER_NAME}}`, `{{VAULT_PATH}}`, etc.) are replaced at install time. To change a value, either re-run `./setup.sh --update` or edit the generated files directly.
